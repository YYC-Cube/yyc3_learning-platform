### Phase 2: 体验深化 — v3.0 → v3.5

**主题**: 学习闭环完善 + 社区互动升级 + 证书系统 + 智能搜索  
**时间**: 2026年3月 — 5月  
**状态**: 📋 规划中

#### 阶段目标

| 目标 | KR | 验收标准 |
|------|-----|---------| 
| 视频课程系统完善 | 真实视频播放 + 进度跟踪 | 断点续播, 观看百分比记录 |
| 社区互动升级 | 评论/回复/私信 | 帖子评论线程 + 实时计数 |
| 证书系统上线 | 自动颁发 + PDF 生成 | 完成模块自动获得证书 |
| 搜索系统强化 | 全站模糊搜索 | 支持课程内容、社区帖子搜索 |
| 个性化推荐 v1 | 基于学习历史 | 推荐命中率 >60% |

#### 具体任务

1. **视频播放系统** (预计 5 天)
   - 集成 Supabase Storage 存储视频资源
   - `LessonPlayer` 对接真实视频 URL (signed URLs)
   - 实现断点续播: `ModuleProgress.lastPosition` 字段
   - 观看完成度自动标记

2. **社区互动增强** (预计 4 天)
   - 数据模型扩展: `PostComment` 类型
   - 评论组件: 嵌套回复 (最多 2 层)
   - @提及用户功能

3. **证书系统** (预计 3 天)
   - 完成模块 → 自动颁发证书
   - PDF 证书生成 (Canvas → Blob → Storage)

4. **搜索系统** (预计 3 天)
   - 服务端跨 KV 前缀搜索
   - 搜索历史 + 热搜聚合

5. **个性化推荐 v1** (预计 3 天)
   - 简化协同过滤算法
   - "为你推荐"仪表盘模块

#### 预期结果

- 学习闭环完整: 观看 → 笔记 → 完成 → 证书
- 社区 DAU 提升，评论互动率 >15%
- 个性化推荐命中率 >60%

## 四、风险评估与缓解

| 风险 | 概率 | 影响 | 缓解策略 | 阶段 |
|------|------|------|---------|------|
| KV 存储性能瓶颈 | 中 | 高 | Phase 4 迁移 PostgreSQL; Phase 2 引入缓存层 | P2-4 |
| Figma Make 环境限制 | 低 | 中 | npm 方案; 纯前端 PDF/Canvas 生成 | P2 |
| 多租户数据隔离安全 | 低 | 高 | RLS + 中间件校验 + 自动化测试 | P3 |
| 支付集成合规风险 | 中 | 高 | Stripe Checkout (PCI 合规) + 法律咨询 | P3 |
| AI API 成本失控 | 中 | 中 | Token 预算; 缓存常见问答; 用户配额 | P4 |
| 直播稳定性 (WebRTC) | 中 | 高 | TURN 备份; Zoom SDK 降级 | P4 |
| 国际化文案质量 | 高 | 中 | Native speaker + AI 翻译 + 社区反馈 | P4-5 |
| 竞品压力 | 中 | 中 | 差异化: 中法双语 + 垂直 + AI | 全周期 |
