## 2. KV 键前缀约定

所有数据通过 KV (Key-Value) 表存储，使用前缀命名约定模拟关系型结构：

| 前缀 | 格式 | 存储类型 | 说明 |
|------|------|---------|------|
| `user:profile:{userId}` | JSON Object | `UserProfile` | 用户完整档案 |
| `user:settings:{userId}` | JSON Object | `UserSettings` | 用户偏好设置 |
| `user:progress:{userId}:{moduleId}` | JSON Object | `ModuleProgress` | 单模块学习进度 |
| `user:achievements:{userId}` | JSON Array | `Achievement[]` | 用户成就勋章列表 |
| `user:activity:{userId}` | JSON Object | `Record<string, number>` | 学习活跃度 (日期→分钟) |
| `user:notes:{userId}` | JSON Object | `Record<string, string>` | 每日学习心得 (日期→文本) |
| `courses_list` | JSON Array | `ModuleCardData[]` | 全部课程列表 |
| `community:posts` | JSON Array | `Post[]` | 社区帖子列表 |
| `kanban:tasks` | JSON Array | `Task[]` | 看板任务列表 |
| `dashboard:stats` | JSON Object | `DashboardStats` | 仪表盘聚合统计 |
| `notes_{lessonId}` | String | `string` | 单课时笔记内容 |

### 3.1 用户档案 (User Profile)

#### GET /user/profile

获取用户档案。

| 参数 | 位置 | 类型 | 默认值 | 说明 |
|------|------|------|--------|------|
| `userId` | query | string | `"default"` | 用户 ID |

**KV Key**: `user:profile:{userId}`

**响应 Data**: `UserProfile | null`

**降级行为**: 前端 `YYC3API.getUserProfile()` 在 API 不可达时返回硬编码的 `defaultProfile`。

---

#### PUT /user/profile

更新用户档案 (合并更新)。

**请求体**: `Partial<UserProfile>`

```json
{
  "id": "default",
  "firstName": "新名称",
  "bio": "更新的个人简介"
}
```

**逻辑**:
1. 读取现有档案 (`kv.get`)
2. 与默认档案、现有档案、请求体三方合并 (`getDefaultProfile → existing → body`)
3. 自动更新 `lastActiveAt` 为当前时间
4. 写回 KV

**响应 Data**: 合并后的完整 `UserProfile`

### 3.3 模块学习进度 (Module Progress)

|------|------|------|--------|

---

#### GET /user/progress/:moduleId

获取特定模块的学习进度。

| 参数 | 位置 | 类型 |
|------|------|------|
| `moduleId` | path | string |
| `userId` | query | string |

**KV Key**: `user:progress:{userId}:{moduleId}`

**响应 Data**: `ModuleProgress | null`

---

#### PUT /user/progress/:moduleId

更新特定模块的学习进度。

| 参数 | 位置 | 类型 |
|------|------|------|
| `moduleId` | path | string |
| `userId` | query | string |

**请求体**: `Partial<ModuleProgress>`

**逻辑**:
1. 读取现有进度
2. 与默认进度结构合并
3. 自动更新 `lastAccessedAt`
4. 写回 KV
5. **自动联动**: 重新计算 `user:profile:{userId}` 的 `completionPercentage` 和 `unlockedModules`

**计算公式**:
```
completionPercentage = (completedCount / totalProgressRecords) * 100
unlockedModules = totalProgressRecords
```

### 4.2 核心函数 `apiFetch<T>`

```typescript
async function apiFetch<T>(
  url: string,
  options?: RequestInit,
  fallback?: T
): Promise<T>
```

**行为**:
1. 发送 HTTP 请求，自动附加 `Authorization` 和 `Content-Type` 头
2. 若响应非 200: 记录错误日志，返回 fallback 或抛出异常
3. 若响应包含标准信封 (`success + data`): 自动解包 `data`
4. 若网络不可达: 捕获异常，返回 fallback 或重新抛出

### 4.3 Fallback 策略

| 方法 | Fallback 值 |
|------|------------|

## 5. 错误处理规范

### 5.1 服务端错误包装

```typescript
const wrap = (label: string, fn: () => Promise<Response>) =>
  fn().catch((err: Error) => {
    console.log(`[ERROR] ${label}: ${err.message}`);
    return new Response(JSON.stringify({
      success: false,
      error: `${label}: ${err.message}`,
      timestamp: new Date().toISOString(),
    }), { status: 500 });
  });
```

所有路由处理函数通过 `wrap()` 包装，确保:
- 错误不会导致服务器崩溃
- 错误信息包含路由上下文 (`label`)
- 统一返回 500 + JSON 错误响应

### 5.2 前端错误处理

```typescript
// apiFetch 内部
if (!res.ok) {
  console.error(`[API ${res.status}] ${url}: ${errorText}`);
  if (fallback !== undefined) return fallback;
  throw new Error(`API request failed: ${res.status}`);
}
```

- 有 fallback: 静默降级，用户无感知
- 无 fallback: 抛出异常，由组件 catch 处理
