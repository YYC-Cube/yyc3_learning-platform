## 1. Project Overview

**YYC3-Learning-Platform** is a commercial SaaS training platform built on Figma Make with:

- **Frontend:** React 18 + Tailwind CSS v4 + Motion (Framer Motion) — Deep Sea Blue theme
- **Backend:** Supabase Edge Functions (Hono web server on Deno)
- **Storage:** KV key-value store with prefix-based namespace (simulates relational data)
- **Auth:** Supabase Auth (email/password + guest mode + token refresh)
- **AI:** BigModel Z.AI SDK integration (GLM-4-Flash free tier, OpenAI, Ollama)
- **Language:** TypeScript strict mode — `any` is prohibited

## 3. KV Key Schema (Critical Reference)

All data is stored in a single `kv_store_dae9c128` PostgreSQL table using prefix-based namespacing:

| KV Key Pattern | Data Shape | Description |
|---|---|---|
| `user:profile:{userId}` | `UserProfile` | User profile (role, avatar, stats) |
| `user:settings:{userId}` | `UserSettings` | Preferences (language, theme, notifications) |
| `user:progress:{userId}:{moduleId}` | `ModuleProgress` | Per-module learning progress |
| `user:achievements:{userId}` | `Achievement[]` | Earned badges/achievements array |
| `user:certificates:{userId}` | `Certificate[]` | Earned certificates array |
| `user:activity:{userId}` | `Record<string, number>` | Heatmap data (date -> minutes) |
| `user:notes:{userId}` | `Record<string, string>` | Daily learning journal |
| `video:progress:{userId}:{moduleId}:{lessonId}` | `VideoProgress` | Video playback position |
| `community:posts` | `Post[]` | All community posts |
| `community:comments:{postId}` | `PostComment[]` | Post comments with nested replies |
| `kanban:tasks` | `Task[]` | Kanban board tasks |
| `dashboard:stats` | `DashboardStats` | Platform-wide aggregate stats |
| `roadmap:sprints` | `RoadmapData` | Sprint phases + progress |
| `sales:records` | `SaleRecord[]` | Sales transaction history |
| `sales:kpis` | `SalesKPI` | Sales KPI aggregates |
| `feedback:{userId}` | `FeedbackEntry[]` | Beta feedback submissions |
| `ai:config:{userId}` | `AIUserSettings` | AI provider/model settings |
| `courses_list` | `ModuleCardData[]` | Course catalog |
| `notes_{lessonId}` | `string` | Per-lesson notes |

## 6. Auth Flow

```
[Login Page]
    |
    +-- Email/Password -> YYC3API.signIn() -> POST /signin
    |   Returns: { user, accessToken, refreshToken }
    |   Stored in: localStorage (yyc3_access_token, yyc3_refresh_token)
    |
    +-- Guest Mode -> enterGuestMode() -> userId='default'
    |
[Session Restore on Mount]
    |
    +-- localStorage.getItem('yyc3_access_token')
    +-- YYC3API.getAuthUser(token) -> GET /auth/me
    +-- If 401: attemptTokenRefresh() -> POST /auth/refresh
    +-- If refresh fails: registerAuthExpiredHandler -> force logout
```

**Key Implementation Details:**
- Token refresh is deduplicated (singleton `_refreshPromise`)
- `apiService.ts` automatically retries once on 401 after successful refresh
- `AuthContext` registers an `authExpiredHandler` callback for forced logout
- Guest mode uses `userId='default'` and bypasses all auth checks

## 7. AI Assistant Architecture (5-Layer)

```
Layer 1: Types          /types/index.ts (AIProvider, AIChatMessage, AIConfigResponse, etc.)
Layer 2: Server Routes  /supabase/functions/server/index.tsx (Routes #41-43)
Layer 3: API Methods    /services/apiService.ts (getAIConfig, saveAIConfig, sendAIChat, sendAIChatStream)
Layer 4: Hook           /hooks/useAIChat.ts (state management, streaming, persistence, retry)
Layer 5: Component      /components/AIAssistant.tsx (3-tab UI, draggable, minimizable capsule)
```

**Provider Priority:** User API key (KV) > Environment variable (`ZHIPU_API_KEY`) > Ollama (local) > Offline fallback

**Streaming Flow:**
1. `useAIChat.sendMessage()` -> `YYC3API.sendAIChatStream()` -> `POST /ai/chat { stream: true }`
2. Server proxies to Z.AI/OpenAI with SSE
3. Hook reads `ReadableStream` chunks, updates `currentStreamText` in real-time
4. On completion, final message is persisted to `localStorage`

**Offline Mode:** When no API key is available, returns canned responses matching keywords (see `OFFLINE_SUGGESTIONS` array in `useAIChat.ts`)

## 13. Coding Conventions

### TypeScript
- **No `any`** — use `unknown` + type guards or explicit interfaces
- All types live in `/types/index.ts` — do not define local types in components
- Use `type` imports: `import type { User } from '../types'`

### Components
- Default avatar: `import { defaultAvatarAsset } from '../services/apiService'`
- Images: Use `<ImageWithFallback>` (from `/components/figma/ImageWithFallback.tsx`) instead of `<img>`
- Animations: Use `motion` package (`import { motion } from 'motion/react'`)
- Toast: `import { toast } from 'sonner@2.0.3'`

### API Calls
- All API calls go through `YYC3API` static class in `/services/apiService.ts`
- Server response envelope: `{ success: boolean, data: T, error?: string, timestamp: string }`
- `apiFetch<T>()` handles unwrapping, fallbacks, and 401 retry automatically

### KV Storage
- All keys use the prefix convention defined in `KV_PREFIXES` (`/types/index.ts`)
- Server-side casting: `kvAs<T>()`, `kvAsArray<T>()`, `kvAsRecord<V>()`
- Error wrapping: `wrap(label, async () => { ... })`

### CSS/Styling
- Tailwind CSS v4 (no `tailwind.config.js`)
- Deep Sea Blue theme: background `#020617`, primary `#3b82f6`
- Font stack: Inter + PingFang SC (Chinese) + system fallbacks
- Protected file: `/styles/globals.css` defines root variables and responsive typography

## 15. Quick Start for New Developers

1. **Read this document** and `/guidelines/Guidelines.md`
2. **Understand the KV schema** (Section 3) — this is your "database"
3. **Trace a feature end-to-end**: e.g., follow `YYC3API.getUserProfile()` from frontend -> `apiService.ts` -> server route -> KV store
4. **Run tests**: `npx vitest run` to verify baseline
5. **Check AI setup**: Set `ZHIPU_API_KEY` in Supabase secrets to enable AI features
6. **Reference existing patterns**: The feedback system (`BetaFeedbackModal` -> `YYC3API.submitFeedback()` -> server route #26 -> KV `feedback:{userId}`) is a clean example of the full stack pattern
